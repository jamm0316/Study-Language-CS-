## 1. 다음 설명에서 괄호에 공통으로 들어갈 알맞은 답을 쓰시오.

파일 구조는 파일을 구성하는 레코드들이 보조기억장치에 편성되는 방식을 의미하는 것으로 크게 순차, (  ), 해싱으로 구분된다.
(  )파일구조는 <값, 주소> 쌍으로 구성되는 데이터 구조를 활용하여 데이터에 접근하는 방식으로, 자기 디스크에 주로 활용된다.

<br><br><br><br><br>

sol) 파일의 구조
1. 순차 파일(Sequential File)
  - 레코드를 논리적인 처리 순서에 따라 연속된 물리적 공간에 기록하는 것을 의미
2. 직접 파일(Direct File)
  - 파일을 구성하는 레코드를 임의의 물리적 저장공간에 기록하는 것
  - 레코드의 특정 기준으로 키가 할당되며, 해싱 함수(Hashing Function)를 이용하여 키에 대한 보조기억장치의 물리적 상대 레코드 주소를 계산한 후 해당하는 주소에 레코드를 저장한다.
    <img width="679" height="245" alt="image" src="https://github.com/user-attachments/assets/4b173a66-9867-4300-a525-f50e46e6a15d" />
3. 색인 순차 파일
  - 색인(인덱스)을 이용한 순차적인 접근 방법을 제공하여 색인 순차 접근 방식이라고한다.
  - 순차 처리와 랜덤 처리가 모두 가능하도록 레코드들의 키 값 순으로 정렬(sort) 시켜 기록하고, 레코드의 키 항목만을 모은 색인 구성하여 편상하는 방식
  - 각 레코드는 키 값 순으로 논리적으로 저장하고, 시스템은 각 레코드의 실제 주소가 저장된 색인을 관리한다.
