## 4. 다음 설명에 해당하는 알맞은 용어를 영문 약어로 쓰시오.
- 다항식 코드를 사용하여 오류를 검출하는 방식이다.
- 동기식 전송에서 주로 사용되며, HDLC 프레임의 FCS(프레임 검사 순서 필드)에 사용된다.
- 접단 오류를 검출할 수 있고 검출률이 높으므로 가장 많이 사용한다.

<br><br><br><br><br>
sol) 오류 검출
- 정의: 데이터 전송 과정에서 발생한 오류를 발견하는 기법
- 핵심: 송신 측이 원본 데이터에 검사용 부가 정보(패리티, CRC 등)를 붙여 전송하고, 수신 측이 같은 규칙으로 재계산하여 값이 다르면 오류로 판단한다. 오류 검출만 하는 방식(ARQ 등으로 재전송 요청)과 오류 정정까지 하는 방식(FEC, 해밍코드 등)으로 나뉜다.

| 방식     | 정의                                         | 핵심                                          | 오류정정 |
| ------ | ------------------------------------------ | ------------------------------------------- | ---- |
| 패리티 비트 | 데이터 비트에 1비트를 추가하여 1의 개수를 짝수/홀수로 맞추는 방식     | 가장 단순, 1비트 오류만 검출 가능. 2비트 이상 오류는 검출 불가      | X    |
| 체크섬    | 데이터를 일정한 단위로 나눠 합산한 값을 전송하여 수신 측에서 비교하는 방식 | IP/TCP 헤더에 사용, 패리티보다 정확하지만 비트 위치는 알 수 없음    | X    |
| CRC    | 다항식(생성 다항식)으ㅏ로 데이터를 나눈 나머지를 검사값으로 사용       | 연속 오류(burst error) 검출에 강함. 데이터링크 계층에서 주로 사용 | X    |
| 해밍코드   | 여러 개의 패리티 비트를 데이터 사이에 삽입하여 오류 위치까지 파악      | 오류 검출 + 1비트 오류 자동 정정 가능. 2^n 위치에 패리티 비트 배치  | O    |
FEC, BEC
