## 13. 접근 통제(Access Control)에 대한 다음 설명에서 괄호에 들어갈 알맞은 용어를 <보기>에서 찾아 쓰시오.

| 구분       | 설명                                                                                                                                              |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| 1️⃣  | • 주체와 객체 등급을 비교하여 접근 권한을 부여하는 방식  <br>• 시스템이 접근 통제 권한을 지정한다.  <br>• 데이터베이스 객체별로 보안 등급을 부여할 수 있다.  <br>• 사영자별로 인가등급을 부여할 수 있다.                   |
| 2️⃣  | • 사용자의 역할에 따라 접근 권한을 부여하는 방식이다.  <br>• 중앙관리자가 접근 통제 권한을 지정한다.  <br>• 임의 접근 통제와 강제 접근 통제의 단점으 보완하였다.  <br>• 다중 프로그래밍 환경에 최적화된 방식이다.              |
| 3️⃣  | • 데이터에 접근하는 사용자의 신원에 따라 접근 권한 부여.  <br>• 데이터 소유자가 접근통제 권한을 지정하고 제어한다.  <br>• 객체를 생성한 사용자가 생성한 객체에 대한 모든 권한을 분여하고, 부여받은 권한을 다른 사용자에게 허가할 수 도 있다. |

sol) Data Access Control(데이터 접근 통제)
접근 통제(Access Control) 3가지 방식

1. MAC (강제 접근 통제)

접근 권한을 **오직 관리자만 부여하고 변경**할 수 있는 방식이다.
각 사용자에게는 **보안 등급(Clearance Level)** 이 부여되고, 각 자원(파일, 데이터 등)에는 **보안 레벨**이 설정된다. 사용자는 자신의 보안 등급이 자원의 보안 레벨 이상일 때만 접근이 허용된다.
사용자가 스스로 권한을 조정하거나 다른 사람에게 권한을 줄 수 없고, 보안 등급·레벨의 수정도 관리자만 가능하다.
> **실제 사용처:** 군사 시스템, 정부 기밀 문서 시스템처럼 철저한 보안 통제가 필요한 환경에서 사용된다. 예를 들어 '일반', '비밀', '극비' 등급으로 문서와 사용자를 분류하고, 등급이 맞지 않으면 관리자라도 임의로 접근을 허용할 수 없다.
<br>

2. DAC (임의적 접근 통제)
접근 권한을 **자원의 소유자가 직접 제어**하는 방식이다.
각 자원에는 **ACL(Access Control List)** 이 존재하며, 소유자는 이 목록을 수정하여 특정 사용자나 그룹에게 권한을 부여하거나 회수할 수 있다. 즉, 관리자가 개입하지 않아도 소유자 판단으로 권한을 조정할 수 있다.
> **실제 사용처:** Linux/Windows의 파일 시스템 권한이 대표적이다. 내가 만든 파일의 읽기·쓰기·실행 권한을 내가 직접 다른 사용자에게 줄 수 있는 구조가 DAC다.
<br>

3. RBAC (역할 기반 접근 통제)
접근 권한을 **사용자 개인이 아닌, 역할(Role) 단위로 관리**하는 방식이다.
관리자는 역할을 정의하고 그 역할에 권한을 묶어둔다. 이후 사용자에게는 역할을 할당하기만 하면 된다. 사용자는 자신에게 할당된 역할이 가진 권한 범위 내에서만 자원에 접근하고 작업을 수행할 수 있다.
> **실제 사용처:** 회사 내부 시스템에서 `개발자`, `HR`, `재무팀` 같은 역할을 만들고, 각 역할에 접근 가능한 메뉴·데이터를 설정해두는 방식이 RBAC다. 신규 입사자가 생기면 개인에게 일일이 권한을 설정하는 게 아니라 역할만 부여하면 된다.
<br>

핵심 차이 요약
| 구분 | 권한 주체 | 기준 |
|---|---|---|
| MAC | 관리자 | 보안 등급·레벨 |
| DAC | 자원 소유자 | 사용자/그룹 신분 |
| RBAC | 관리자 (역할 설계) | 역할(Role) |
